v5.4.0 亮点

LLVM 后端自举编译打通：原生二进制无 Go 依赖 🔥

2026-07-22 发布。自举源码 src/*.klx（7 文件 5250 行）经 LLVM 后端编译成原生二进制 kylix_self_llvm（127KB，无 Go 依赖），推进 KylixRT 里程碑。IR 生成成功（736KB）→ llc 通过 → 链接成原生二进制 → 运行 exit 0 产出 Go 代码（含真换行 + WriteLn 语句识别 + is/as 运行时类型分派）。本轮修复 20+ 个 LLVM 后端缺口：类型系统、全局变量、record 支持、外部方法、is/as 运行时 RTTI、map 值类型化、builtin 等。教程通过率 51/51 (100%)。

🗺️

map[String]Variant 真实化

htab 值槽类型无关——存 box 指针 vs C 字符串对 htab 透明。v5.1 让 map[String]Variant 的值槽装 Variant box：emitMapIndexGet 走 htab_get_variant 返回 "variant"，emitMapIndexPut 装箱 RHS。cache/string-map 不回归。

m['pi'] = 3.14 → variant_compare
🔧

htab_get_variant + nilbox

新 htab_get_variant：miss 返回 @__kylix_variant_nilbox（tag=0 全局）→ as_* 走 nil 默认（str→""、int→0、float→0.0、bool→false），与「missing key 返回默认」契约一致。受 needVariantRuntime 守卫。

JsonDecodeMap → 真实 Variant map
➕

Variant 算术

variant_add（either str→拼接；both int→int；else double）、variant_sub/mul（both int→int else double）、variant_div（始终 double）。coerceValue 加 variant→concrete，n := v 解箱。仅 LLVM（Go interface{} 不支持运算符）。

variant_add(ptr a, ptr b) → ptr
📤

JsonGet* 全部 unbox

JsonGetString/Int/Float/Bool 经 variant_as_str/int/double/bool 解箱；JsonGetMap/Array unbox raw 子串再 parse。新增 as_int/as_bool 主体。双端 parity。

转译对比

Pascal → Go，一目了然

Kylix 将 Pascal 源码编译为干净、可读、可直接运行的 Go 代码。没有黑魔法，没有运行时依赖。LLVM 后端则直接生成原生二进制。

Pascal 源码 (.klx) 
program WebApp;
uses web;

var app: TServer;
begin
  app := web.createServer(8080);

  app.get('/hello',
    procedure(req: TRequest; res: TResponse)
    begin
      res.json(record
        message := 'Hello, Kylix!';
      end);
    end);

  app.listen();
end.
生成的 Go 代码 (.go) 
package main

import "kylix/stdlib"

var app *stdlib.TServer

func main() {
  app = stdlib.CreateServer(8080)

  app.Get("/hello",
    func(req *stdlib.TRequest, res *stdlib.TResponse) {
      res.Json(struct {
        Message string
      }{Message: "Hello, Kylix!"})
    })

  app.Listen()
}

KylixBoot 框架

声明式 Web 开发，类似 Spring Boot

KylixBoot 把路由、依赖注入、中间件、优雅停机封装成一站式框架。23 单元测试、~700 行运行时，uses boot 即可使用。

全局快捷方式 (函数式) 
program Quick;
uses boot;

begin
  boot.GET('/users/:id',
    procedure(req: TRequest; res: TResponse)
    begin
      res.JSON(record
        id := req.Param('id');
        name := 'Alice';
      end);
    end);

  boot.POST('/users',
    procedure(req: TRequest; res: TResponse)
    begin
      res.Status(201).JSON(req.Body());
    end);

  // 内置中间件
  boot.Use(boot.Logger());
  boot.Use(boot.Recover());
  boot.Use(boot.CORS());
  boot.Use(boot.RateLimit(100, 60));

  // 优雅停机内置 (SIGTERM/SIGINT)
  boot.Listen(':8080');
end.
声明式 (注解 + DI 容器) 
[Controller('/api/users')]
type
  TUserController = class
    [Inject]
    UserRepo: TUserRepository;

    [Get('/')]
    function ListUsers(req: TRequest): TResponse;
    begin
      result := req.JSON(UserRepo.FindAll());
    end;

    [Post('/'), Authenticated]
    function CreateUser(req: TRequest): TResponse;
    begin
      var user := req.Body<TUser>();
      UserRepo.Save(user);
      result := req.Created(user);
    end;
  end;

[Entity('users')]
type
  TUser = class
    [Column('id'), PrimaryKey]
    Id: Integer;
    [Required, Email]
    Email: String;
  end;

kylix build --target=darwin/arm64     # 交叉编译 (5 个原生目标)

kylix run --keep                      # 保留生成的 .go 文件

kylix test                            # 运行测试 (*_test.klx, Setup/Teardown)
kylix test --filter Add               # 按名称过滤
kylix bench                           # 性能基准测试

kylix doc                             # 生成 Markdown 文档 (含代码示例)

# 包管理
kylix add github.com/user/pkg         # 添加依赖 (git / 本地路径)
kylix install                         # 安装锁定的依赖
kylix remove pkg                      # 移除依赖

# 信息 & 国际化

编译后端

一份源码，多个目标

Kylix 源代码可以编译到 5 种平台：Go 二进制、LLVM 原生、WebAssembly、WASI 和自举编译器。所有目标共享同一个 AST 与类型系统。

🐹

Go 后端（默认）

Pascal → Go 源码 → 原生二进制。利用 Go 内置的交叉编译，目标机器无需安装运行时。

kylix build --target=linux/arm64
⚙️

LLVM 原生后端

AST → LLVM IR → 目标文件 → 原生二进制。绕过 Go 工具链，支持 -O0/O1/O2/O3 优化。M2 Phase 1 已完成数组 + 优化。

kylix build --backend=llvm --llvm-opt=3
🌐

WebAssembly

编译到 wasm 在浏览器或 Node.js 运行。Go 标准（~3 MB）或 TinyGo（~30 KB）二选一。

kylix build --wasm --tinygo
🪐

WASI 运行时

Wasmtime、Wasmer、Cloudflare Workers 跨平台执行。完整的 stdin/stdout/env/args 支持。

kylix build --wasi main.klx
♻️

自举编译器（Kylix 写 Kylix）

7 个 .klx 源文件可重新编译自身。Diff 验证：Go 参考实现 vs Kylix 自举 — 语义等价。

token / lexer / parser / generator
📦

包注册中心

Registry 服务端（REST API + htmx 前端）。kylix publish 发布，kylix add 安装。即将部署到 packages.kylix.top。

SQLite / PostgreSQL 存储 
# 同一份 hello.klx，五种目标
kylix build hello.klx                          # Go → 本地原生
kylix build --target=darwin/arm64 hello.klx    # → macOS Apple Silicon
kylix build --target=linux/amd64 hello.klx     # → Linux x86-64
kylix build --target=windows/amd64 hello.klx   # → Windows x86-64

kylix build --wasi hello.klx                   # WASI (Wasmtime)
